# 信号跟踪系统设计

**设计时间**: 2026-04-22  
**版本**: 2.2 (带自动跟踪)

---

## 🎯 设计目标

1. **防止重复触发**: 每小时跑一次，同一天只触发一次
2. **自动跟踪结果**: 信号触发后自动跟踪后续表现
3. **自动止盈止损**: 达到目标自动结束信号
4. **数据积累**: 为自我进化提供准确数据

---

## 📊 去重逻辑

### 现有设计 ✅

```python
alert_key = f"{bond_code}_latent_{today}"  # 使用当天日期
if alert_key not in history.get('alerts', []):
    # 触发信号
```

**优点**:
- ✅ 同一天只触发一次
- ✅ 每小时跑也不会重复
- ✅ 第二天可以重新触发（如果信号还在）

**示例**:
```
09:30 运行 → 检测到信号 → 记录 alert_key="123456_latent_2026-04-22"
10:30 运行 → alert_key 已存在 → 跳过
11:30 运行 → alert_key 已存在 → 跳过
次日 09:30 → alert_key="123456_latent_2026-04-23" → 可以触发
```

## 🔄 信号跟踪流程

### 1. 信号触发时

```python
# 保存到数据库
db.save_signal({
    'bond_code': '123456',
    'bond_name': 'XX 转债',
    'stock_code': '300XXX',
    'signal_date': '2026-04-22',
    'signal_type': 'latent',
    'stock_quality': {'rating': 'B', 'score': 75},
    'days_since_tongguo': 32,
})
```

### 2. 每次监控时自动跟踪

```python
# 获取活跃信号（未结束的）
active_signals = tracker.get_active_signals()

for signal in active_signals:
    # 获取当前股价
    prices = sina.fetch_history(stock_code, days=10)
    
    # 计算收益
    current_return = (current_price - signal_price) / signal_price * 100
    
    # 检查结束条件
    if hold_days > 30:
        status = 'ended', reason = '持有超过 30 天'
    elif current_return <= -5:
        status = 'ended', reason = '止损 -5%'
    elif current_return >= 10:
        status = 'ended', reason = '止盈 +10%'
    elif '同意注册' in progress and hold_days >= 2:
        status = 'ended', reason = '同意注册后卖出'
```

### 3. 信号结束时

```python
# 保存结果到数据库
db.save_outcome({
    'bond_code': '123456',
    'signal_date': '2026-04-22',
    'exit_date': '2026-04-25',
    'entry_price': 48.57,
    'exit_price': 50.50,
    'return': +3.97,
    'success': True,
    'hold_days': 3,
    'exit_reason': '止盈 +10%',
    'stock_quality': {'rating': 'B', 'score': 75},
    'days_since_tongguo': 32,
})
```

## 📁 核心模块

### lib/signal_tracker.py

**类**: `SignalTracker`

**方法**:
- `get_active_signals()` - 获取活跃信号
- `check_signal_status(signal)` - 检查单个信号状态
- `update_all_signals()` - 更新所有信号
- `get_tracking_report()` - 生成跟踪报告

**结束条件**:
| 条件 | 阈值 | 说明 |
|------|------|------|
| 持有时间 | >30 天 | 超时自动结束 |
| 止损 | ≤-5% | 亏损 5% 止损 |
| 止盈 | ≥+10% | 盈利 10% 止盈 |
| 同意注册 | 持有≥2 天 | 公告后卖出 |

## 📊 数据流

```
监控运行
  ↓
检测新信号
  ↓
保存到 signals_history.json
  ↓
下次监控
  ↓
检查活跃信号
  ↓
获取当前股价
  ↓

  ↓
```

## ⚠️ 注意事项

1. **定时运行**: 建议开市时间每小时跑一次
2. **去重逻辑**: 使用日期作为 key，同一天只触发一次
3. **跟踪频率**: 每次监控都会自动跟踪活跃信号
4. **数据备份**: 定期备份 `data/` 目录
5. **手动修正**: 可以手动修改 outcomes 数据
